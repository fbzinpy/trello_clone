Cheatcode Frontend — Trello Clone

Guía de estudio para preguntas de lógica frontend en vivo. El objetivo no es memorizar todo: es saber dónde tocar, qué patrón usar y cómo no romper el flujo.

1. Flujo sagrado del frontend

USUARIO HACE ALGO ↓ COMPONENTE captura click / input / submit / drag ↓ VALIDACION o LOGICA local ↓ EMIT al padre o llamada API ↓ ESTADO reactivo cambia ↓ VUE repinta la pantalla
Regla madre
  • Si cambia lo visual, mira el <template> o el <style>.
  • Si cambia una condicion, validacion o comportamiento, mira el <script setup>.
  • Si un componente hijo avisa algo al padre, busca emit(...).
  • Si el padre le pasa datos al hijo, busca props o atributos como :boards, :board, :col.
  • Si hay backend, casi siempre pasa por api desde axios.js.

2. Mapa mental de archivos

Entrada y rutas

  • main.js: monta Vue, Pinia y Router.
  • App.vue: contiene <router-view />.
  • router/index.js: rutas y proteccion con token.
  • store/auth.js: token, usuario y logout.
  • axios.js: baseURL e interceptor JWT.

Vistas

  • Login.vue: login, registro, mensajes y redireccion.
  • Dashboard.vue: centro de control de secciones, busqueda, tableros y modales.
  • Precios.vue: pagina publica estatica.

Tableros y Kanban

  • BoardGrid.vue: pinta tarjetas de tablero y menu.
  • CreateBoardModal.vue: formulario para crear tablero.
  • BoardModal.vue: tablero abierto, columnas y tarjetas.
  • KanbanColumn.vue: una columna.
  • KanbanCard.vue: una tarjeta.

Logica reutilizable

  • useBoards.js: cargar, crear, duplicar y borrar tableros.
  • useDragDrop.js: mover tarjetas entre columnas.
  • AppHeader.vue: buscador, crear, perfil, tema y logout.
  • AppSidebar.vue: cambiar seccion activa.

4. Patrones de logica frontend

Patron 1

Validar antes de ejecutar

Usalo cuando: pidan minimo de caracteres, campo obligatorio, no permitir vacios, evitar duplicados.

function submit() {
  const value = input.value.trim()

  if (!value) {
    error.value = 'Campo requerido'
    return
  }

  if (value.length < 3) {
    error.value = 'Minimo 3 caracteres'
    return
  }

  emit('create', value)
}

Archivos tipicos: Login.vue, CreateBoardModal.vue, KanbanColumn.vue, BoardModal.vue.

Patron 2

Confirmar antes de borrar

Usalo cuando: pidan confirmar eliminar tablero, columna o tarjeta.

function handleDelete(item) {
  const ok = confirm(`Eliminar "${item.name}"?`)
  if (!ok) return

  deleteItem(item.id)
}

Idea clave: el return corta la funcion si el usuario cancela.

Patron 3

Filtrar listas

Usalo cuando: pidan buscar por nombre, mostrar solo algunos tableros o filtrar segun condicion.

function filterBoards(boards, term) {
  const normalizedTerm = term.trim().toLowerCase()

  if (!normalizedTerm) return boards

  return boards.filter((board) =>
    board.name.toLowerCase().includes(normalizedTerm)
  )
}

Archivo clave: Dashboard.vue.

Patron 4

Ordenar sin romper el estado original

Usalo cuando: pidan ordenar tableros alfabeticamente o por cantidad.

const sortedBoards = computed(() => {
  return [...userBoards.value].sort((a, b) =>
    a.name.localeCompare(b.name)
  )
})

Idea clave: [...array] crea copia. Asi no mutas el arreglo original sin querer.

Patron 5

Limitar cantidad

Usalo cuando: pidan maximo de tableros, columnas o tarjetas.

function handleCreateBoard(name) {
  if (userBoards.value.length >= 5) {
    alert('Maximo 5 tableros')
    return
  }

  createBoard(name)
}

Donde mirar: si el limite es de tableros, Dashboard.vue o useBoards.js. Si es de tarjetas/columnas, BoardModal.vue.

Patron 6

Cambiar donde se inserta algo

Usalo cuando: pidan que algo aparezca al inicio en vez de al final.

// Al final
lista.push(item)

// Al inicio
lista.unshift(item)

Ejemplos: columna nueva al inicio, tarjeta nueva al inicio, tablero nuevo al inicio.

Patron 7

Mover y revertir si falla

Usalo cuando: pidan cambiar logica de drag/drop o manejar error de API.

// 1. mover localmente para que la UI responda rapido
const [card] = fromCol.cards.splice(idx, 1)
toCol.cards.push(card)

try {
  // 2. persistir en backend
  await api.patch(`/boards/cards/${card.id}/move`, { columnId: toCol.id })
} catch (e) {
  // 3. revertir si falla
  toCol.cards.pop()
  fromCol.cards.splice(idx, 0, card)
  alert('No se pudo mover la tarjeta')
}

Archivo clave: useDragDrop.js.

5. Props, emits y estado: lo que mas preguntan

Props

Padre → hijo

El padre le pasa datos al hijo.

<BoardGrid
  :boards="filteredUserBoards"
  :show-add="true"
/>

En el hijo:

defineProps({
  boards: { type: Array, required: true }
})
Emit

Hijo → padre

El hijo no decide todo. Avisa al padre que algo paso.

// Hijo
emit('delete-board', board)

// Padre
@delete-board="handleDeleteBoard"
Ref

Estado editable

Para valores que cambian: inputs, modal abierto, seccion activa.

const search = ref('')
search.value = ''
Computed

Estado derivado

Para valores calculados desde otros estados.

const filteredBoards = computed(() =>
  filterBoards(userBoards.value, search.value)
)

Caso

const name = boardName.value.trim()

if (name.length < 3) {

Caso

Confirmar antes de eliminar tablero

Archivo: Dashboard.vue

Funcion: handleDeleteBoard(board)

const ok = confirm(`Eliminar el tablero "${board.name}"?`)
if (!ok) return
Caso

Hacer privada la ruta de precios

Archivo: router/index.js

{ path: '/precios', component: Precios, meta: { requiresAuth: true } }

Ya existe el beforeEach que redirige si no hay token.

Caso

Si ya hay sesion, mandar de login a dashboard

Archivo: router/index.js

router.beforeEach((to) => {
  const auth = useAuthStore()

  if (to.path === '/' && auth.token) {
    return '/dashboard'
  }

  if (to.meta.requiresAuth && !auth.token) {
    return '/'
  }
})
Caso

Limpiar buscador al cambiar de seccion

Archivo: Dashboard.vue

En vez de asignar directo desde el template, creas una funcion:

function selectSection(section) {
  selectedSection.value = section
  search.value = ''
}

Y en el template usas:

@select-section="selectSection"
Caso

Tarjeta nueva al inicio de la columna

Archivo: BoardModal.vue

Funcion: addCard

// Cambiar esto
col.cards.push(data)

// Por esto
col.cards.unshift(data)
Caso

Tarjeta movida al inicio de columna destino

Archivo: useDragDrop.js

Funcion: onDrop

// Cambiar esto
toColObj.cards.push(card)

// Por esto
toColObj.cards.unshift(card)

Si haces esto, tambien debes ajustar el revert: en vez de pop(), quitar del inicio con shift().

7. Checklist para responder en vivo

  1. Escucha la palabra clave: login, ruta, tablero, columna, tarjeta, drag, buscador, tema.
  2. Abre el archivo dueño usando la tabla de esta guia.
  3. Decide si es template, script setup o style.
  4. Busca el handler: submit, handle..., add..., delete..., onDrop.
  5. Aplica el patron: validar, confirmar, filtrar, ordenar, limitar, insertar o revertir.
  6. Cambia lo minimo y prueba.

8. Mantra de parcial frontend

Si el profe pide logica, casi siempre va en <script setup>.

Si pide mostrar u ocultar algo, busco v-if, v-show o clases dinamicas.

Si pide cambiar una lista, busco filter, sort, push, unshift o splice.

Si pide comunicar componentes, pienso en props para bajar datos y emit para subir eventos.

Si pide API, busco api.get, api.post, api.patch o api.delete.

Si me pierdo, vuelvo al flujo: usuario hace algo → handler → estado cambia → Vue repinta.
